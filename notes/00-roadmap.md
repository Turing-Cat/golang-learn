# Go 学习路线图

> 创建日期：2026-04-02

## 学习档案

| 项目 | 情况 |
|------|------|
| 编程基础 | 有 Python 简单脚本经验，了解变量、函数、条件、循环 |
| Go 基础 | 零基础 |
| 学习目标 | 找工作/面试 |
| 每日时间 | 30 分钟 |
| 学习风格 | 偏代码实践 |

## 推荐起点

有一定编程基础（变量、函数、循环等概念），跳过"编程共通知识补课"，从 Go 环境搭建开始。重点关注 Go 与 Python 的差异：静态类型、零值、包管理、错误处理等。

## 四个阶段

### 阶段 1：Go 基础语法与程序结构（约 3-4 周）

| 小节 | 重点 | 难点 | 注意（和 Python 的差异） |
|------|------|------|------|
| 1.1 Hello World / package / import | 程序入口和包机制 | package 和 import 的规则 | Python 没有强制包声明 |
| 1.2 变量、常量、类型、零值 | 静态类型系统 | 零值概念 | Python 是动态类型 |
| 1.3 输入输出与格式化 | fmt 包 | 格式化动词 | 和 Python f-string 不同 |
| 1.4 条件与循环 | if/switch/for | Go 只有 for 一种循环 | Python 有 for/while |
| 1.5 函数、多返回值 | 函数是一等公民 | 多返回值、命名返回值 | Python 用 tuple 返回 |
| 1.6 作用域 | 块作用域 | 短变量声明的作用域 | 和 Python 类似但更严格 |
| 1.7 数组、切片、map | 切片是核心 | 切片 vs 数组、底层数组共享 | Python list/dict 的差异 |
| 1.8 基础整合练习 | 综合运用 | 全部串联 | 查缺补漏 |

### 阶段 2：Go 核心抽象与语言特性（约 3-4 周）

| 小节 | 重点 | 难点 |
|------|------|------|
| 2.1 struct | Go 没有 class | 用 struct 组织数据 |
| 2.2 method & receiver | 给类型绑定行为 | 值接收者 vs 指针接收者 |
| 2.3 指针 | 理解地址和引用 | 指针的直觉和使用时机 |
| 2.4 interface | Go 风格的多态 | 隐式实现、空接口 |
| 2.5 error 处理 | Go 的错误哲学 | if err != nil 模式 |
| 2.6 defer/panic/recover | 资源清理 | 只在真正需要时用 panic |
| 2.7 包与模块 | 工程组织 | go mod、导出规则 |
| 2.8 代码组织练习 | 分文件写项目 | 从单文件过渡到多文件 |

### 阶段 3：工程能力与标准库（约 4-5 周）

| 小节 | 重点 | 难点 |
|------|------|------|
| 3.1 文件读写 & io | 标准库 io | Reader/Writer 接口 |
| 3.2 JSON | encoding/json | 结构体标签、编解码 |
| 3.3 测试 | testing 包 | 表驱动测试 |
| 3.4 goroutine & channel | 并发核心 | 并发思维 |
| 3.5 sync 并发安全 | Mutex、WaitGroup | 数据竞争 |
| 3.6 context | 取消与超时 | 上下文传递 |
| 3.7 HTTP | net/http | 写服务和客户端 |
| 3.8 配置、日志、项目结构 | 工程实践 | 分层组织 |
| 3.9 数据库基础 | database/sql | CRUD + 错误处理 |

### 阶段 4：项目实战（约 3-4 周）

| 项目 | 目标 |
|------|------|
| CLI 工具 | flag/参数解析、文件操作、命令行输出 |
| REST API | HTTP 路由、JSON、中间件、测试 |
| 分层后端项目 | handler/service/repository 分层 |

## 每周节奏（每天 30 分钟）

- **周一至周四**：学新概念 + 写代码练习（每天 1 个小知识点）
- **周五**：做本周综合练习
- **周六**：整理笔记 + 错题 + 复习
- **周日**：休息或自由探索

## 第一周计划

| 天     | 内容                    |
| ----- | --------------------- |
| Day 1 | Go 环境搭建 + Hello World |
| Day 2 | 变量、常量、基本类型            |
| Day 3 | 格式化打印                 |
| Day 4 | 条件与循环                 |
| Day 5 | 综合小练习                 |
| Day 6 | 整理笔记 + 复习             |

## 过关标准

每节结束时用"概念解释 + 代码检查 + 变式检查"中的至少 2 项验收：
1. 能用自己的话说清概念
2. 能独立写出可运行代码
3. 换个条件也能做出来

## 补基础机制

1. 换讲法（换类比、简化例子）
2. 拆小题（分 2-4 个微步骤）
3. 补前置知识
4. 补基础回路（连续失败时暂停，做专项补课）
5. 学习策略调整（练习为主、填空题、具象场景等）
